Setting up for Qt

General requirements

Compiling Qt based applications and libraries involves steps that are executed before the build. Namely, MOS, UI, resource and translation compilers have to be run in order to generate files that are used during the build process. These processes are added to a pre-build step for every Qt based project and use Lua to call the respective compilers. So having Lua installed is a common dependency, regardless of the host platform.

Qt configuration on Windows

Here are the steps needed to use build with Qt on Windows:

  • Install Qt using the online installer
    Alternatively, Qt can be built from source.

  • Install Lua for Windows

  • Set environment variable that points to the installation of Qt
    The variable looks like this: QTDIR_VS2017_x64
    Middle part represents GENie action and suffix represents architecture (x86 or x64)
    It shoud be set to point to a root directory of specific Qt version, for example: C:\Qt\5.11.2\msvc2017_64

Qt configuration on Linux

Linux setup is similar to Windows:

  • Install Qt using the online installer

  • Install Lua and lfs:
    sudo apt-get install luarocks
    sudo luarocks install luafilesystem

  • Add Qt PKG_CONFIG path to ~/.bashrc
    export PKG_CONFIG_PATH=$PKG_CONFIG_PATH:/home/USER/Qt/5.11.2/gcc_64/lib/pkgconfig

  • Install qt5-default
    sudo apt-get install qt5-default

That’s all!
