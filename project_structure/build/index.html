build

Root directory of build contains the main scripts. The entry point for all projects is the build.lua script and it contains utility functions and loads the rest of the scripts needed to generate projects/solutions. Here is the breakdown of each script and its purpose:

  • build.lua
    This is the main script that is to be loaded by every project/solution script. It contains all the utility functions and loads the rest of the scripts needed to set everything up.

  • configurations.lua
    This script runs for each combination of platform/configuration and configures project settings, compiler and linker switches, etc.

  • deploy.lua
    WIP: platform specific deployment code is located here. It servers to package executable, icons and similar resources in a way that is required by a target platform.

  • embedded_files.lua and embedded_shader_prebuild.lua
    WIP: source file embedding and shader (bgfx) processing.

  • project_XXX.lua
    All scripts with project_ prefix handle a specific project type.

  • qtprebuild.lua
    Script that is run by Qt based projects before each build. Handles MOC, UIC, QRC, etc.

  • qtpresets5
    Qt 5 project configuration presets, used to prepare Qt based project for builds

  • toolchain
    Based on scripts made for bgfx with modifications tailored for build. This script configures toolchains for every target platform/compiler.
